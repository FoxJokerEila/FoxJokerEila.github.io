第二章 let 和 const 命令

2.1 let 命令

2.1.1 基本用法

  • let 只在相应代码块里有效

  • 在循环中 let 声明的变量 在每一轮循环中 都相当于一个新的变量 之所以可以根据上一轮循环中变量的值进行计算 是因为 JS引擎内部 记住了 上一轮的变量的值 所以可以计算

  • for循环比较特殊：设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域

  • 假如在循环中对一个变量赋值了一个函数，然后在循环外调用了这个函数，那程序就会跑回对应的某轮循环产生的代码块里面去 执行(相当于每次循环都生成了一个代码块)

2.1.2 不存在变量提升

  • var 在声明变量前就可以使用 值为 undefined

  • let不允许这样 会报错

2.1.3 暂时性死区 / 在声明前不准使用

  • 假设var声明了一个变量，在一个代码块中，首先对该变量进行了赋值，然后用 let声明了一个同名的局部变量，这样就会报错

  • 因为 一旦 用let对该变量进行了声明 该变量就会绑定这个代码块，由于let不允许变量提升 所以之前的赋值代码 会报错

    • x 被赋值为 y ，但是 y 此时还没有声明，所以会报错

2.1.4 不允许重复声明

  • let 不允许在相同作用域内重复声明同一个变量

2.2 块级作用域

2.2.1 为什么需要块级作用域

  • 内层变量 发生 变量提升 导致 内层变量 覆盖 外层变量

  • 可能会变量泄露，作为循环变量在循环结束时没有消失，而是泄露成了全局变量

2.2.2 ES6 的块级作用域

  • let 实际上为 JS 新增了块级作用域。

  • 外层作用域无法读取内层作用域的变量。

  • 内层作用域可以定义外层作用域的同名变量，并且互不影响。

  • 块级作用域的出现使立即执行匿名函数不再必要了(?)

2.2.3 块级作用域与函数声明

  • ES5规定函数只能声明在顶层作用域和函数作用域之内，不能在块级作用域中声明

  • ES6 明确允许在块级作用域中声明函数，在块级作用域中 函数声明语句的行为类似于 let ，在块级作用域之外不可引用

  • 在外部已经声明了的函数 如果在块级作用域内又声明了一次 即使该代码不会执行 也会报错

    • 以上代码中出现了 立即执行函数、立即执行函数中的 if 语句、立即执行函数中的 if 语句中的函数声明
    • 一般来说函数声明的提前会击穿 if 语句 ，但这里的立即执行函数中的 函数声明会自动转化为函数表达式 也就是 var f = function(){}
    • 根据变量提升，提升的部分是var f,函数体不会被提升，所以f的值是undefined

    • 需要注意的是 在普通函数中的函数提升是提升整个函数的, 立即执行函数只提升 var f 部分

  • 函数声明类似于 var,即会提升到全局作用域或函数作用域的头部

  • 同时，函数声明还会提升到所在的块级作用域的头部

    • var color = "red"会被分解为 var color和color = "red"，其中var color会被提升到函数体的顶端，所以color的值不是 blue 而是 未定义的 undefined

  • 注意，ES6 的块级作用域允许声明函数的规则只在使用大括号的情况下成立，如果没用就会报错

2.3 const 命令

2.3.1 基本用法

  • 使用const的时候必须给变量初始化。
  • const的作用于和let一样，只在对应的块级作用域内有效。
  • 存在变量提升。。不过书里说不存在，不过确实存在暂时性死区，只能在声明后使用。学姐说，不能称之为提升，应该是 预处理 ，存在一个 TDZ ，相当于只是声明了但没有赋值
  • 不能用const声明已存在的变量。

2.3.2 本质

  • const保证的是变量指向的那个内存地址不得改动。

  • 对于简单类型来说，值就存在那个内存地址中。

  • 对于复合类型的数据(对象和数组)来说，变量指向的内存地址中保存的是一个指针。

  • 这样的话，给其内部的数据进行修改添加删除都可以执行。但是直接对其本身进行赋值是不可以的。

  • 如果想真正冻结一个对象-应使用 Object.freeze 方法

    • 常规模式下 对冻结了的对象的数据进行操作 改代码不会生效
    • 严格模式下会报错

    • 递归的方法进行冻结

2.3.3 ES6 声明变量的 6 种方法

  • ES5 只有 var 和 function
  • ES6 添加了 let 和 const ，以及 import 和 class

2.4 顶层对象

  • 顶层对象在浏览器中指的是谁 window 对象

  • 在 Node 环境中指的是 global对象

  • 在 ES5 中，顶层对象的属性与全局变量是等价的(这被认为是设计败笔)

    • 无法在编译时提示变量未声明
    • 程序员可能无意之中创建全局变量
    • 顶层对象的属性在任何地方都可以读写，者不利于模块化编程
    • 另一方面 window指的是浏览器窗口对象，这样不合适

  • var和function命令声明的全局变量属于顶层对象的属性

  • let``const``class命令声明的全局变量不属于顶层对象的属性。

  • 也就是说，从ES6开始，全局变量将逐步与顶层对象的属性隔离

2.5 global 对象

ES5 的顶层对象本身就是个问题，因为它在各种实现中不统一

  • 浏览器中是 window ，但 Node 和 Web Worker 没有 window
  • 在浏览器中和 Web Worker 中， self 也指向顶层对象，但是 Node 没有 self
  • 在 Node 中，顶层对象是 global ，但其他环境都不支持

  • 有个方案解决了这个问题

  • 在所有环境下，global都是存在的，都可以拿到顶层对象

  • 练习案例见 JavaScriptBasis/Chapter2/提案解决方案取到global.js

第三章 变量的解构赋值

3.1 数组的解构赋值

3.1.1 基本用法

  • 概念：ES6 允许按照一定模式从数组和对象中提取值，然后对变量进行赋值，这被称为解构 (Destructuring)。

  • 以前，为变量赋值只能直接指定值。let a = 1

  • 现在可以这样：let [a, b, c] = [1, 2, 3]

  • ‘从数组中提取值，按照对应位置对变量赋值’

  • 本质上，这种写法属于“模式匹配”，只要等号两边的模式相同，左边的变量就会被赋予对应的值。

  • 支持嵌套 let [foo, [[bar], baz]] = [1, [[2], 3]]

  • 一些例子：

  • 另一种情况是不完全解构，即等号左边的模式只匹配一部分的等号右边的数组。这种情况依然可以成功。

  • 解构不成功的情况会报错，如果等号右边不是数组或者可遍历的解构。

    • ```javascript
      // 报错
      let [foo] = 1 // 值或转化为对象后不具备 Iterator 接口
      let [foo] = false // 值或转化为对象后不具备 Iterator 接口
      let [foo] = NaN // 值或转化为对象后不具备 Iterator 接口
      let [foo] = null // 值或转化为对象后不具备 Iterator 接口
      let [foo] = {} // 本身不具备 Iterator 接口

  • ES6内部使用严格相等运算符===判断一个位置是否有值。如果一个数组成员不严格等于 undefined ，默认值是不会生效的

  • ```javascript
    let [x = 1] = [undefined] // x = 1
    let [x = 1] = [null] // x = null

    • 最后一个是 暂时性死区

3.2 对象的解构赋值

  • 对象的解构和数组不同，数组变量的取值由它的位置决定

  • 而对象的属性没有次序，变量必须与属性同名才能取到正确的值

  • ```javascript
    let { bar, foo } = { foo: “aaa”, bar: “bbb” }
    foo // “aaa”
    bar // “bbb”

    let { baz } = { foo: “aaa”, bar: “bbb” }
    baz // “undefined”

  • 对象的解构赋值的内部机制是先找到同名属性，然后再赋值给对应的变量。真正被赋值的是后者，而不是前者

  • 可以见得，我们第一个例子实际上是下面代码的简写：

  • ```javascript
    let { foo: foo, bar: bar } = { foo: “aaa”, bar: “bbb” }

  • var node = {
  loc: {
    start: {
      line: 1,
      colums: 5
    }
  }
}

var { loc, loc: { start }, loc: { start: { line } } } = node;

loc // { start: { line: 1, colums: 5 } }
start.colums // 5
line // 1

### 3.6 圆括号问题

  • ES6规则：只要有可能导致解构的歧义，就不得使用圆括号

    • let {( x: c )} = {}

    • let {(x): c} = {}

  • 函数参数

    • 函数参数也属于变量声明

  • 赋值语句的模式

3.6.2 可以使用圆括号的情况

  • 赋值语句的非模式部分可以使用圆括号
  • 以上，都不是声明语句，他们的圆括号都不属于模式的一部分。

第四章 字符串扩展

  • ES6加强了对 Unicode 的支持，并且扩展了字符串对象。

4.1 字符的 Unicode 表示法

  • \uxxxx 表示一个字符 ，xxxx表示字符的 Unicode 码点
  • 仅限于 \u0000 ~ \uFFFF 超出范围则必须使用 2 个 双字节的形式表达 "\uD842\UDFB" // "𠮷"
  • 如果超过了 0xFFFF 则 JS 会理解成 \u20BB + 7 。 由于 \u20BB 是一个不可打印的字符，所以只会显示一个空格，后面跟一个 7
  • ES6 改进了这一点，将码点放入大括号就能正确解读该字符。
  • "\u{20BB7} // "𠮷"
  • "\u{41}\u{42}\u{43}" // "ABC"

4.2 codePointAt()

  • JS内部，每个字符占 2 个字节，如果是需要四个字节储存的字符(Unicode 码点大于 0xFFFF 的字符)，JS会认为是 2 个字符。

  • 为了读取完整的四字节字符，我们使用 codePointAt 方法。

  • var s = '𠮷a'

s.codePointAt(0) // 134071
s.codePointAt(1) // 57271
s.codePointAt(2) // 97

4.3 String.fromCodePoint()

  • ES5提供了String.fromCharCode方法，用于从码点返回对应字符，但是这个方法不能识别 32 位的 UTF-16 字符

    • String.fromCharCode(0x20BB7)
    • // 一个奇怪的字符
    • 因为不能识别 所以舍弃了最高的两位，输出了 U+0BB7 对应的字符

  • ES6提供了 String.fromCodePoint 方法，可以识别大于 0xFFFF 的字符，弥补了 ES5 的不足。

    • String.fromCodePoint(0x20BB7)
    • // "𠮷"
    • 如果有多个参数 则会被合并为一个字符串返回

  • 注意：String.fromCodePoint定义在 String对象上，而codePointAt定义在字符串的实例对象上

4.4 字符串的遍历器接口

  • ES6 为字符串添加的遍历器接口可以识别大于 0xFFFF 的码点，传统的 for 循环无法识别这样的码点。

    • var text = String.fromCodePoint(0x20BB7)

for (let codePoint of 'text') {
    console.log(codePoint)
}

// "𠮷"

### 4.6 normalize()

  • Unicode 用两种方式表示语调符号和重音符号

    • 直接提供字符 \u01D1
    • 提供合成字符\u004F\u030C
    • 但是 JS 无法识别他们的等价
    • '\u01d1' === '\u004F\u030C' // false
    • 因为在 JS 眼中他们长度不同
    • '\u01D1'.length // 1
    • '\u004F\u030C'.length // 2

  • ES6 为字符串实例提供了一个 normalize 方法，可以将字符的不同表示方法统一为同样的形式，这称为 Unicode 正规化

    • '\u01D1'.normalize() === '\u004F\u030C'.normalize() // true

  • normalize 可以接受一个参数 决定运作方式

    • NFC 合成字符

    • NFD 分解字符

  • normalize 方法目前不能识别三个及以上字符的合成。

4.7 includes()、startsWith()、endsWith()

  • 传统上，JS 中只有 indexOf 方法可用来确定一个字符串是否包含在另一个字符串内。

  • ES6 又提供了三种方法。

    • includes(): 返回布尔值，表示是否找到了参数字符串
    • startsWith(): 返回布尔值，表示参数字符串是否在源字符串的头部
    • endsWith(): 返回布尔值，表示参数字符串是否在源字符串的尾部

  • 三个方法都支持第二个参数，表示开始搜索的位置

    • endsWith 特别一点 它针对的是前 n 个字符，其他两个是从 n 开始到结束的字符。

4.8 repeat()

  • repeat 方法返回一个新字符串，表示将源字符串重复n次

  • 如果参数为小数，则取整

  • 如果是负数或者是 Infinity 则报错

  • 如果是 介于 -1 ~ 0 之间的小数，则等同于0，因为进行了取整运算，视同为0

  • NaN 等同于 0

  • 如果参数是字符串，则会先转化成数字

4.9 padStart()、padEnd()

  • padStart()用于补全头部
  • padEnd()用于补全尾部
  • 接受两个参数。一个用来指定字符串的最小长度，第二个是用来补全的字符串
  • 如果字符串长度大于指定长度，返回原字符串
  • 如果补全字符串+原字符串长度超过指定长度，则会截取超出位数的补全字符串
  • 如果省略第二个参数，则会用空格来补全
  • 用途
    • 补全指定尾数
    • 生成提示字符串 如日期
