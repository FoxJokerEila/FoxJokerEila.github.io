vue-cli脚手架

最新版的脚手架项目目录中已经没有build和config文件夹了，webpack部分的配置被整合到可选的vue.config.js文件中，需要自己配置。

  • static文件夹中的内容会原封不动的被复制到dist文件夹中
  • .babelrc文件规定语法转换等配置
  • .editorconfig文件规定代码语法格式，自动首行缩进，尾部加空行
  • .eslintignore忽略某些地方的不规范的语法，如build/config/dist等等
  • .gitignore忽略上传一些文件，node_modules/dist/.log文件等等

使用

  • 导入路由对象，调用Vue.router(VueRouter)
  • 创建路由实例，传入路由映射配置
  • 在Vue实例中挂载创建的路由实例

在router文件夹下的index.js导出

在main.js中导入（只写一个router目录，会自动寻找index.js文件）

  • <router-lin>：该标签是一个vue-router已经内置的组件，它会被渲染成一个标签
  • <router-view>：该标签会根据当前的路径，动态渲染出不同的组件
    • 网页的其他内容，比如顶部的导航，或者底部的一些版权信息会和<router-view>处于同一个等级
    • 在路由切换时，切换的是<router-view>挂载的组件，其他内容不会发生改变

router-link的其他属性

tag：可以指定将<router-link>之后渲染成什么组件，比如tag=”li”，就会渲染成一个

  • 元素

    replace：不会留下history记录，后退键返回不能返回到上一个页面中

    active-class：当<router-link>对应的路由匹配成功时，会自动给当前元素设置一个router-link-active的class，设置active-class可以修改默认的名称。

    • 自己写的业务代码放在了app.js中

    vue-router-路由懒加载

    • 路由中通常会定义很多不同的页面
    • 如果只是打包在一个js文件中
    • 会导致这个页面非常庞大
    • 如果一次性从服务器请求下来这个页面需要花费很长时间
    • 此时我们使用懒加载
    • 路由懒加载的主要作用就是将路由对应的组件打包成一个个的js代码块
    • 只有在这个路由被访问到的时候，才加载对应组件

    懒加载的方式

    认识嵌套路由

    • 创建对应的子组件，并在路由映射中配置对应的子路由
    • 在组件内部使用<router-view>标签

    vue-router-参数传递

    传递参数主要有两种类型：params和query

    • params的类型：

      • 配置路由格式：/rouuter/:id
      • 传递的方式：在path后面跟上对应的值
      • 传递后形成的路径：/router/123，/router/abc

    • query的类型：

      • 配置路由格式：/router，也就是普通配置
      • 传递的方式：对象中使用query的key作为传递方式
      • 传递后形成的路径：/router?id=123，/router/?id=abc
