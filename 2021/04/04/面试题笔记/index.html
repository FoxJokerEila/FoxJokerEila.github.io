4.4 重排和重绘

浏览器渲染页面的步骤是：

  1. 加载DOM元素->DOM tree
  2. 加载CSS部分->Style struct
  3. 生成Render树->DOM + CSS = render
  4. paint

重排：当页面的元素的影响布局的属性发生变化时会触发重排 reflow

重绘：当页面的属性（不影响布局）改变时发生重绘 repaint

共同点：都是通知render来进行的。

4.16 元素水平垂直居中的方法

居中元素（子元素）的宽高已知

  • 利用定位+margin:auto:

    • 子绝父相
    • 上下左右定位的值都设为0，如果此时子元素没有设置宽高的话就会被拉开到和父级一样宽高
    • 如果子元素设置了定位，子元素的虚拟占位也是撑满了父元素，此时再给一个margin: auto就可以上下左右都居中了

  • 利用定位+margin:负值：

    • 子绝父相
    • 子元素top、left设为50%
    • 子元素margin-left和margin-top设为子元素宽度的一半

居中元素宽高未知

  • 利用定位+transform

    • 子绝父相
    • 子元素top、left设为50%
    • 子元素transform: translate(-50%, -50%);
    • 不需要知道自身的宽高

  • table布局

    • 父元素设置display: table-cell、vertical-align: center、text-align: center
    • 子元素设置display: inline-block

  • flex弹性布局

    • 父元素display: flex、主轴：align-items: center、副轴：justify-content: center
    • 子元素不用管

  • grid网格布局

    • 父元素display: grid、主轴：align-items: center、副轴：justify-content: center
    • 子元素不用管

4.16 BFC

BFC（Block Formatting Context），即块级格式化上下文，它是页面中的一块渲染区域，有一套属于自己的渲染规则：

  • 内部的盒子会在垂直方向上一个接一个的放置

  • 对于同一个BFC的两个相邻的盒子的margin会发生重叠，与方向无关

  • 每个元素的外边距与包含块的左边界相接触（从左到右），即是浮动元素也是如此

  • BFC的区域不会与float的元素区域重叠

  • 计算BFC的高度时，浮动的子元素也参与计算

  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然

BFC目的是形成一个相对于外界完全独立的空间，让内部的子元素不会影响到外部的元素

触发条件

  • 根元素，即HTML元素
  • 浮动元素：float值为left、right
  • overflow值不为visible，为auto、scroll、hidden
  • display值为inline-block、inline-flex、grid、inline-grid
  • position的值为absolute或fixed

应用场景

  • 放置margin重叠（塌陷）

  • 清除内部浮动

    • <style>
    .par {
        border: 5px solid #fcc;
        width: 300px;
    }

    .child {
        border: 5px solid #f66;
        width:100px;
        height: 100px;
        float: left;
    }
</style>
<body>
    <div class="par">
        <div class="child"></div>
        <div class="child"></div>
    </div>
</body>

理解：通过伪元素绝对定位到行位并遮住文字，再通过`overflow: hidden`隐藏多余文字

    优点：

    • 兼容性好，对各大主流浏览器有好的支持
    • 响应式截断，根据不同宽度做出调整

    注意：当文本存在英文的时候，可以设置word-break: break-all使一个单词能够在换行时进行拆分

  • 基于行数截断

    可以看到，上述使用了webkit的css属性扩展，所以兼容浏览器范围是pc端的webkit内核的浏览器，由于移动端大多数是使用webkit，所以移动端常用该形式

    注意：如果文本为一段很长的英文或者数字，则需要添加word-wrap: break-word属性

4.23 让Chrome支持小于12px的文字方式有哪些？区别？

Chrome中文版浏览器会默认设定页面的最小字号是12px，英文版没有限制

因为Chrome团队认为汉字小于12px就会增加识别难度

  • 中文版浏览器

    与网页语言无关，取决于用户在Chrome的设置里

  • 系统级最小字号

    浏览器默认设定页面的最小字号，用户可以前往chrome://settings/fonts根据需求更改

    而我们在实际项目中不能奢求用户改变浏览器设置

解决方案：

  • zoom
  • -webkit-transform: scale()
  • -webkit-text-size-adjust: none

    zoom的意思是变焦，可以改变页面上元素的尺寸，属于真实尺寸

    其支持的值类型有：

4.25 CSS预编语言

1. 是什么：

css语言作为一门标记性语言，语法相对简单，对使用者的要求较低，但同时也带来了一些问题

需要书写大量看似没有逻辑的代码，不方便维护及扩展，不利于复用，尤其对于非前端开发工程师来讲，往往会因为缺少 Css 编写经验而很难写出组织良好且易于维护的 Css 代码

预处理语言：

扩充了css语言，增加了诸如变量、混合(mixin)、函数等功能，让css更易维护、方便

本质上，预处理是css的超集

包含一套自定义的语法和一个解析器，根据这些语法定义自己的样式规则，这些规则则最终会通过解析器，编译生成对应的css文件

2. 有哪些

  • sass
  • less
  • stylus

sass

​ 受less影响，已经进化到了全面兼容css的scss

​ 文件后缀名为.sass与scss，可以严格按照sass的缩进方式省去大括号和分号

less

​ 受sass的影响较大，但又使用css的语法，更容易上手

​ 而且影响sass演变到了scss时代

stylus

​ stylue是一种预处理框架，2010年产生，来自Node.js社区，主要用来给Node项目进行css预处理机制

3. 区别

  • 变量（variables）
  • 作用域（scope）
  • 代码混合（mixins）
  • 嵌套（nested rules）
  • 代码模块化（Modules）
